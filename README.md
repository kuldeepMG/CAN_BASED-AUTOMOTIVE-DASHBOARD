🚗 From Sensors to Dashboard – Developed a CAN-Based Automotive System



💡 Every time we look at a vehicle dashboard, we see real-time information such as speed, RPM, gear position, and turn indicators. But have you ever wondered how this data is collected from different parts of the vehicle and displayed so accurately?



To explore this, I developed a CAN-Based Automotive Dashboard system using the PIC18F4580 microcontroller and the Controller Area Network (CAN) protocol.



The system consists of three Electronic Control Units (ECUs) communicating over a CAN network:

⚙️ ECU 1 – Transmits speed and indicator status.

⚙️ ECU 2 – Transmits RPM and gear data.

⚙️ ECU 3 – Receives CAN messages and displays dashboard information



Throughout this project, I worked with:

• CAN communication protocol

• ADC for sensor data acquisition

• Digital keypad interfacing

• CLCD display interfacing

• Timer interrupts

• Embedded C programming on PIC18F4580



This project provided hands-on experience in automotive communication systems, real-time embedded firmware development, and peripheral interfacing using industry-relevant protocols.
